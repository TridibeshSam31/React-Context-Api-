React Context API is a state-sharing mechanism.

Where Context Is Perfect

Use Context ONLY for data that is:

‚úîÔ∏è 1. Needed in many components
‚úîÔ∏è 2. Not frequently changing
‚úîÔ∏è 3. Not huge in size

Examples:

Cart count (not full cart items)



Where Context Is a BAD Idea (Important)

Don‚Äôt use Context for:

Rapidly changing data (typing, animations, frequent API polling)
Large lists (products, messages, posts)
Local state that only 1‚Äì2 components need

Why?

Because every state update inside Context re-renders ALL consumers.
That kills performance.



Creating Context
import { createContext } from "react";

export const AuthContext = createContext();


Creating a Provider Component
export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);

  const login = (data) => setUser(data);
  const logout = () => setUser(null);

  return (
    <AuthContext.Provider value={{ user, login, logout }}>
      {children}
    </AuthContext.Provider>
  );
}

Using It
import { useContext } from "react";
import { AuthContext } from "../context/AuthContext";

function Dashboard() {
  const { user, logout } = useContext(AuthContext);

  return (
    <>
      <h1>Welcome {user?.name}</h1>
      <button onClick={logout}>Logout</button>
    </>
  );
}
